In this assignment, you will implement a closed set face identification model using the Olivetti faces dataset. The goal is to create a neural network that can recognize faces from a known set of individuals and add new individuals to the set if they are not already present.

What is Closed Set Face Identification?

Closed set face identification is the task of identifying a face from a known set of individuals. In a closed set, every query face is assumed to belong to one of the known identities. The system needs to match the query face to one of these known identities. In some scenarios, like this assignment, we also add the capability to identify if a face does not belong to any known identity and add it to the set.

Cosine Similarity in Face Identification

Cosine similarity is a measure used to determine how similar two vectors are, regardless of their magnitude. In face identification, it is used to compare the similarity between feature vectors extracted from images of faces. Here's a concise explanation:

  1. Feature Extraction: Each face image is processed by a neural network to extract a feature vector, which is a high-dimensional representation of the face's unique characteristics.

  2. Cosine Similarity Calculation: Cosine similarity measures the cosine of the angle between two feature vectors. The formula is:

    $$ \text{Cosine Similarity} = \frac{\mathbf{A} \cdot \mathbf{B}}{\|\mathbf{A}\| \|\mathbf{B}\|} $$

    where $\mathbf{A}$ and $\mathbf{B}$ are the feature vectors, $\mathbf{A} \cdot \mathbf{B}$ is the dot product, and $\|\mathbf{A}\|$ and $\|\mathbf{B}\|$ are the magnitudes (or norms) of the vectors.

  3. Range and Interpretation: The cosine similarity score ranges from -1 to 1.

    • 1: The vectors are identical (i.e., the faces are the same).
    • 0: The vectors are orthogonal (i.e., there is no similarity between the faces).
    • -1: The vectors are diametrically opposed (not typically used in face identification since face features don't usually yield negative similarity).

  4. Usage in Face Identification: In practice, if the cosine similarity between the feature vectors of two face images exceeds a certain threshold, the faces are considered to belong to the same person. This method is effective because it focuses on the orientation of the vectors, making it robust to variations in lighting, scale, and facial expressions.

By using cosine similarity, face identification systems can efficiently and accurately match and recognize faces based on their feature representations.

Explanation of train_faces, gallery_faces, and eval_faces

In the context of our face recognition task using the Olivetti faces dataset, it's important to understand the roles of different subsets of data: train_faces, gallery_faces, and eval_faces. Each of these subsets serves a distinct purpose in training and evaluating our face recognition model.

train_faces

  • Purpose: This subset of the dataset is used to train the model.
  • Content: It contains the faces of 6 unique individuals. These images are used by the model to learn the features and patterns that distinguish one person's face from another.
  • Usage: During the training process

gallery_faces

  • Purpose: This subset is used as a reference set during evaluation. It's sometimes called the "gallery set."
  • Content: It contains half of the images of 4 other individuals (different from those in train_faces). These individuals are not seen by the model during training.
  • Usage: The model generates embeddings (feature representations) for these faces after it has been trained with train_faces. These embeddings are stored and later used to compare against the embeddings of faces in the eval_faces set. The labels for these faces are also stored to help in the identification process.

eval_faces

  • Purpose: This subset is used to test the model's performance. It's also known as the "evaluation set."
  • Content: It contains the other half of the images of the same 4 individuals found in the gallery_faces set. These images are also not used during training.
  • Usage: After training, the model generates embeddings for these faces. The goal is to match these embeddings to the closest embeddings in the gallery_faces set using a similarity measure, such as cosine similarity. This helps determine how well the model can identify individuals it wasn't trained on, based solely on the reference gallery.

Summary

  • train_faces: Used for training the model; contains faces of 6 individuals.
  • gallery_faces: Used as a reference set during evaluation; contains half the images of 4 different individuals.
  • eval_faces: Used to test the model's performance; contains the other half of the images of the same 4 individuals in the gallery_faces set.

def train_model(train_faces_tensor, train_labels_tensor, num_epochs=20):
    train_dataset = TensorDataset(train_faces_tensor, train_labels_tensor)
    train_loader = DataLoader(train_dataset, batch_size=32, shuffle=True)
    model = FaceEmbeddingModel()

    criterion = nn.CrossEntropyLoss()
    optimizer = optim.Adam(model.parameters(), lr=0.001)

    for epoch in range(num_epochs):
        running_loss = 0.0
        for i, (inputs, labels) in enumerate(train_loader):
            optimizer.zero_grad()

            outputs = model(inputs)
            loss = criterion(outputs, labels)
            loss.backward()
            optimizer.step()

            running_loss += loss.item()
            if i % 10 == 9:  # Print every 10 mini-batches
                print(f'Epoch [{epoch + 1}/{num_epochs}], Step [{i + 1}/{len(train_loader)}], Loss: {running_loss / 10:.4f}')
                running_loss = 0.0
    return model

from sklearn.metrics.pairwise import cosine_similarity

def evaluate_model(model, eval_faces_tensor, eval_labels_tensor, gallery_embeddings_loaded, gallery_labels_loaded):
    model.eval()
    with torch.no_grad():
        eval_embeddings = model(eval_faces_tensor).numpy()

    similarity_matrix = cosine_similarity(eval_embeddings, gallery_embeddings_loaded)

    similarity_indices = np.argmax(similarity_matrix, axis=1)
    predicted_labels = gallery_labels_loaded[similarity_indices]

    accuracy = np.mean(predicted_labels == eval_labels_tensor.numpy())
    print(f'Accuracy: {accuracy * 100:.2f}%')

    return predicted_labels, similarity_indices
